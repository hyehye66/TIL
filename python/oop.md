`왼쪽 : 절차지향, 오른쪽 : 객체지향` 

* 객체지향은 주어,동사 구조처럼 생각하면 됨
* class : 종류
  * Camel Case로 씀 ; 클래스 이름
* 인스턴스 메서드
  * self : 인스턴스 자기자신/ 매개변수 이름으로 첫번째 인자로 정의/ 다른 단어로 써도 되지만 암묵적인 규칙
  * snake_case로 씀 ; 함수, 변수 이름

* 인스턴스와 클래스 간의 이름 공간(namespace)
  * 클래스 : 정의 순간
* static 메서드: 자동으로 넘어가지 않음
* 정리
